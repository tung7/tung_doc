Layout

  • The layout tag

nidium supports various layout operations similar to what Web browser offer, though with many differences.

The layout tag

Within an .nml file, the layout tag is pretty similar to the <body> tag -- a container for the application layout (at initial state).

Examples

Using the layout tag to create an element

<application>
    <meta>
        <title>nidium</title>
        <viewport>200x150</viewport>
        <identifier>com.nidium.app</identifier>
    </meta>
    <assets>
        <script>
            /* Lookup for an existing element within the document */
            var elem = document.getCanvasById("main");

            /* Get a 2d context on the new canvas */
            var ctx = elem.getContext("2d");

            /* Draw a red square */
            ctx.fillStyle = "red";
            ctx.fillRect(0, 0, 50, 50);

        </script>
    </assets>

    <layout>
        <canvas id="main" width="50" height="50"></canvas>
    </layout>
</application>

As you can see, there is no need for an onload event, even though the script is inserted before the layout

Achieve the same result, creating the element programmaticaly (without using the layout syntax)

<application>
    <meta>
        <title>nidium</title>
        <viewport>200x150</viewport>
        <identifier>com.nidium.app</identifier>
    </meta>
    <assets>
        <script>
            /* Create a new canvas of size 50x50 */
            var elem = new Canvas(50, 50);

            /* Add the newly created canvas as a child to the root element */
            document.canvas.add(elem);

            /* Get a 2d context on the new canvas */
            var ctx = elem.getContext("2d");

            /* Draw a red square */
            ctx.fillStyle = "red";
            ctx.fillRect(0, 0, 50, 50);

        </script>
    </assets>
</application>
